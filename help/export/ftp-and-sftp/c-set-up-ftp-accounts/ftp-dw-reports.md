# Rapports distribués sur l’Data Warehouse

>[!NOTE]
>
>Les informations suivantes concernent les types de destinations FTP et SFTP. FTP et SFTP sont des types de destination hérités. Lors de la configuration d’un rapport Data Warehouse, vous devez utiliser un type de destination cloud, qui est plus sécurisé. Pour plus d’informations sur la configuration des types de destination cloud pour un rapport Data Warehouse, voir [Création d’une requête Data Warehouse](/help/export/data-warehouse/create-request/t-dw-create-request.md).

Data Warehouse vous permet de demander des rapports qui présentent un niveau avancé de relations entre les données brutes, en fonction de vos questions.

Dans ce cas, la distribution FTP est disponible dans le cadre de la configuration des [!UICONTROL options de remise avancées]. Pour utiliser le protocole FTP, vous avez besoin d’un compte FTP interne existant ou d’un compte FTP Adobe. Adobe ne configure pas automatiquement le compte FTP en votre nom par l’intermédiaire de l’interface de l’entrepôt de données. La diffusion FTP doit être immédiate, selon la taille du fichier et le nombre de rapports traités.

Pour plus d’informations sur les limites FTP et la conservation des données, voir [Limites FTP et conservation des données](/help/export/ftp-and-sftp/ftp-limits.md).
