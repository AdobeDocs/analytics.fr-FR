# Termes utilisés dans Adobe Analytics

Utilisez ce glossaire pour comprendre le contexte de nombreux termes utilisés par Adobe Analytics.

* **Activity Map :** Module complémentaire de navigateur montrant les zones sur lesquelles l'utilisateur a cliqué le plus souvent sur votre site. Voir [Carte d'activités](../analyze/activity-map/activity-map.md) dans le Guide de l'utilisateur d'Analytics.
* **Console d'administration :** Peut faire référence à :
   * Outils d'administration hérités, où les paramètres d'une suite de rapports dans Adobe Analytics sont gérés. Dans les versions précédentes d'Adobe Analytics, les autorisations utilisateur étaient également gérées ici. Voir Outils [d'administration](../admin/admin/c-admin-tools.md) dans le guide de l'utilisateur Admin.
   * La console d'administration Adobe, où l'accès aux produits est configuré et les permissions utilisateur sont gérées. Voir Console [d'administration](../admin/admin-console/home.md) dans le guide de l'utilisateur Admin.
* **Affectation :** Si une variable de conversion rencontre plusieurs valeurs lors d'une visite, le paramètre d'attribution de la variable détermine quelle valeur est conservée. See [Conversion variables](../admin/admin/conversion-var-admin/conversion-var-admin.md) in the Admin user guide.
* **Anomalie :** Détection à l'aide de la modélisation statistique pour détecter automatiquement les tendances imprévues dans les données. Le modèle analyse les mesures et détermine une limite inférieure, une limite supérieure et une plage de valeurs attendues. Voir [Détection des anomalies](../analyze/analysis-workspace/virtual-analyst/c-anomaly-detection/anomaly-detection.md) dans le guide de l'utilisateur Analyser.
* **Appmeasurement :** Bibliothèque de code utilisée pour collecter des données et les envoyer à Adobe. Voir [la page d'accueil](../implement/home.md) du Guide de l'utilisateur Mise en œuvre.
* **Emplacement ASI :** Il n'existe plus. Dans les versions précédentes d'Adobe Analytics, les emplacements ASI fournissaient un conteneur de suites de rapports temporaire pour afficher les données segmentées. Dans la version actuelle d'Adobe Analytics, les segments peuvent être appliqués instantanément à n'importe quel rapport.
* **Ventilation :** Vous permet d'afficher une dimension dans le contexte d'une autre dimension. Voir [Ventilation des dimensions](../analyze/analysis-workspace/components/dimensions/t-breakdown-fa.md) dans le guide de l'utilisateur Analyser.
* **Rebond :** Visite comprenant un accès unique. Voir [Rebonds](../components/c-variables/c-metrics/metrics-bounces.md) dans le guide de l'utilisateur des composants. Voir aussi Accès unique.
* **Mesure calculée :** Permet la combinaison de mesures, de fonctions statistiques et de formules existantes pour les rapports. Voir [Mesures calculées](../components/c-calcmetrics/cm-overview.md) dans le guide de l'utilisateur Composants.
* **Campagne :** Peut faire référence à :
   * Variable Campaign qui remplit la dimension Code de suivi. Voir [Variables de page](../implement/js-implementation/c-variables/page-variables.md) dans le Guide de l'utilisateur Implémentation.
   * Classification par défaut de la dimension Code de suivi ; créées automatiquement pour toutes les suites de rapports.
   * Adobe Campaign, composant d'Adobe Experience Cloud. Plus d'informations sur [Adobe.com](https://www.adobe.com/marketing/campaign.html).
* **Canal :** Peut faire référence à :
   * Variable de canal qui renseigne la dimension Sections du site. Voir [Variables de page](../implement/js-implementation/c-variables/page-variables.md) dans le Guide de l'utilisateur Implémentation.
   * Canaux marketing, composant qui aide à comprendre comment les utilisateurs arrivent sur votre site. Voir [Canaux marketing](../components/c-marketing-channels/c-overview.md) dans le guide de l'utilisateur des composants.
* **Classification :** Fonctionnalité d'Adobe Analytics qui permet le regroupement des valeurs de dimension. Voir [Classifications](../components/c-classifications2/c-classifications.md) dans le guide de l'utilisateur des composants.
* **Clickmap :** Plus utilisé. Un module complémentaire hérité montre les zones sur lesquelles a été cliquée le plus grand nombre de visiteurs sur votre site. Cet outil a été abandonné en faveur de Carte d'activités.
* **Flux de données Parcours de navigation :** Voir Flux de données.
* **Cohorte :** Groupe de personnes partageant des caractéristiques communes au cours d'une période donnée. See [What is Cohort Analysis?](../analyze/analysis-workspace/visualizations/cohort-table/cohort-analysis.md) dans le guide de l'utilisateur Analyser.
* **Serveur de collecte :** Voir Serveur de collecte de données.
* **Variables de données contextuelles :** Variables temporaires utilisées uniquement dans les règles de traitement. Les valeurs de variable de données contextuelles sont définitivement perdues si une règle de traitement ne les copie pas dans une variable de conversion ou de trafic. Voir [Variables de données contextuelles](../implement/js-implementation/c-variables/context-data-variables.md) dans le Guide de l'utilisateur Implémentation.
* **Variable de conversion :** Evars connues sous le nom d'evars. Stocke une valeur personnalisée et conserve la valeur de la variable jusqu'à son expiration. See [Conversion variables](../components/c-variables/dimensionslist/reports-conversion.md) in the Components user guide.
* **Corrélation :** Plus utilisé comme terme ; remplacé par des ventilations de dimension. Dans les versions précédentes d'Adobe Analytics, les corrélations permettaient de ventiler les variables de trafic. Voir [Ventilation des dimensions](../analyze/analysis-workspace/components/dimensions/t-breakdown-fa.md) dans le guide de l'utilisateur Analyser.
* **Données actives :** Option de certains rapports qui permet l'inclusion de données récemment collectées qui n'ont pas encore été entièrement traitées. Voir Données [actives](../analyze/reports-analytics/current-data.md) dans le Guide de l'utilisateur d'Analytics.
* **Lien personnalisé :** Type d'accès contenant des données non vues. Voir la fonction [s. tl ()](../implement/js-implementation/function-tl.md) dans le guide de l'utilisateur Implémentation. Voir aussi Accès.
* **Attributs du client :** Fonctionnalité Experience Cloud qui permet le transfert de données d'attributs. Voir Attributs [du client](https://docs.adobe.com/content/help/en/core-services/interface/customer-attributes/attributes.html) dans le guide de l'utilisateur des services principaux.
* **Délégué du service clientèle :** Un utilisateur désigné autorisé à interagir directement avec le service à la clientèle Adobe. Voir [Délégués à l'assistance clientèle](https://helpx.adobe.com/experience-cloud/supported-users.html) dans la base de connaissances de Experience Cloud.
* **Serveur de collecte de données :** Serveurs détenus par Adobe qui reçoivent et traitent des données. Les demandes d'image sont envoyées aux serveurs de collecte de données d'Adobe pour utilisation dans les rapports.
* **Connecteurs de données :** Solution de développement complète qui permet à un tiers d'automatiser le transfert des données dans Adobe Analytics. Les clients de ce tiers peuvent utiliser un connecteur de données pour enrichir leurs données dans Adobe Analytics. La plupart des connecteurs de données utilisent un processus similaire utilisé dans les sources de données. Reportez-vous à la section Connecteurs de données dans le guide de l'utilisateur Importer.
* **Flux de données :** Exportation de données brutes qui répertorie chaque accès comme une ligne et des variables comme des colonnes distinctes. Le plus souvent utilisé pour exporter des données Adobe Analytics vers une base de données tierce. Voir Flux [de données](../export/analytics-data-feed/c-getstarted/data-feed-overview.md) dans le guide de l'utilisateur Exporter.
* **Sources de données :** Permet à un utilisateur de télécharger des données à partir d'un fichier dans Adobe Analytics. Le fichier est généralement extrait d'un site FTP. Voir [Sources de données](../import/c-data-sources/datasrc-home.md) dans le guide de l'utilisateur Importer.
* **Entrepôt de données :** Fonctionnalité d'Adobe Analytics qui vous permet de demander des rapports plus volumineux. Voir [Entrepôt de données](../export/data-warehouse/data-warehouse.md) dans le guide de l'utilisateur Exporter.
* **Dimension :** Type de composant contenant des valeurs de variable, comme le texte. notamment le Nom de page, le Code de suivi ou le Domaine référent. Une mesure est généralement son équivalent.
* **Gestion dynamique des balises :** Ancienne solution de gestion des balises d'Adobe. Voir [Présentation de l'implémentation de la gestion dynamique des balises](../implement/c-implement-with-dtm/dtm-implementation-overview.md) dans le Guide de l'utilisateur Implémentation. Adobe recommande d'utiliser plutôt Adobe Experience Platform Launch.
* **Sérialisation d'événements :** Processus consistant à implémenter des mesures pour empêcher la collecte d'événements en double. Voir [Sérialisation d'événements](../implement/js-implementation/event-serialization.md) dans le Guide de l'utilisateur Implémentation.
* **Evar :** Voir Variable de conversion.
* **Événement :** Voir Evénement de réussite.
* **Excelclient :** Plus utilisé comme terme. Nom du prédécesseur du créateur de rapports.
* **Expiration :** Dans le contexte d'une variable de conversion, durée pendant laquelle la valeur persiste sur le serveur principal. Cette persistance permet d'associer les événements aux valeurs de variable avant l'accès de l'événement. See [Conversion variables](../admin/admin/conversion-var-admin/conversion-var-admin.md) in the Admin user guide.
* **Flux :** Type de visualisation dans Analysis Workspace qui montre les chemins empruntés par les utilisateurs sur votre site. Voir Visualisation [du flux](../analyze/analysis-workspace/visualizations/c-flow/flow.md) dans le guide de l'utilisateur Analyser.
* **Genesis :** Plus utilisé comme terme. Ancien nom des connecteurs de données.
* **Suite de rapports globale :** Terme informel désigné pour une suite de rapports qui collecte les accès à partir de plusieurs sites.
* **Code H :** Prédécesseur d'appmeasurement. Dans les versions précédentes d'Adobe Analytics, les versions de code étaient mesurées par « version H », comme H .27 .5, H .26, etc.
* **Accès :** Une demande d'image unique envoyée aux serveurs de collecte de données Adobe. Les pages vues et les liens personnalisés peuvent être appelés accès.
* **Demande d'image :** Image transparente de 1 x 1 pixel utilisée pour communiquer avec les serveurs de collecte de données Adobe. Un site Web demande cette image invisible avec une longue chaîne de requête contenant des données ; Adobe renvoie l'image invisible et analyse la chaîne de requête reçue.
* **Insight :** Peut faire référence à :
   * Ancien nom de Data Workbench.
   * Custom Insight, un nom historique pour la variable de trafic personnalisée.
* **IPC :** Abréviation pour l'indicateur de performance clé. Mesures qui aident une entreprise à comprendre les performances de son site. Chaque organisation possède des indicateurs clés de performance différents qui mesurent différents aspects de son activité. Voir [Création d'un document de conception de solution](../implement/prepare/solution-design.md) dans le Guide de l'utilisateur Implémentation.
* **Latence :** délai entre la collecte des données et le moment où elles sont disponibles dans les rapports. La latence type d'une suite de rapports est de 30-90 minutes. Voir [Latence](../technotes/latency.md) dans le guide de l'utilisateur Technotes.
* **Lancement :** Short for Adobe Experience Platform Launch, Adobe's Current Implementation Solution. Voir [Présentation](https://docs.adobe.com/content/help/en/launch/using/overview.html) du guide de l'utilisateur de lancement d'Adobe Experience Platform.
* **Prop de liste :** Paramètre qui convertit une variable de trafic type en prise en charge de plusieurs valeurs dans le même accès. Toute variable de trafic personnalisée peut devenir une propriété de liste si le paramètre est activé. Voir [Variables de page](../implement/js-implementation/c-variables/page-variables.md) dans le Guide de l'utilisateur Implémentation.
* **List var :** Variable distincte distincte des variables de conversion. Les variables de liste prennent en charge plusieurs valeurs dans le même accès et les valeurs de variable sont conservées lors d'une visite, comme dans le cas des variables de conversion. Seuls trois variables de liste sont disponibles pour une organisation. Voir [Variables de page](../implement/js-implementation/c-variables/page-variables.md) dans le Guide de l'utilisateur Implémentation.
* **Connexion à la société :** Collection de suites de rapports utilisées par votre organisation. Certaines organisations sont dotées de plusieurs sociétés de connexion qui s'appliquent à différentes parties de leur organisation.
* **Canal marketing :** Fonctionnalité d'Adobe Analytics qui classe les accès par mode d'arrivée sur votre site. La logique utilisée pour classer les accès peut être personnalisée à l'aide des règles de traitement des canaux marketing. Voir [Prise en main des canaux marketing](../components/c-marketing-channels/c-getting-started-mchannel.md) dans le guide de l'utilisateur des composants.
* **Mesure :** Type de composant contenant des données quantitatives. Les valeurs de mesure contiennent généralement des nombres, tels que Pages vues, Visites et Recettes. Une dimension est généralement son équivalent.
* **Balisage multi-suite :** Pratique d'envoi du même accès à plusieurs suites de rapports. Avec l'introduction aux suites de rapports virtuelles, cette pratique n'est plus nécessaire. La plupart des actions de balisage multi-suite permettent d'intégrer une suite de rapports globale.
* **Normalisation :** Moyen d'organiser une visualisation qui prend toutes les mesures et les force à des proportions égales, ce qui permet une comparaison plus facile des tendances.
* **Occurrences :** Type de mesure qui indique le nombre d'accès qu'une valeur de dimension a été définie ou conservée. Voir [Occurrences](../components/c-variables/c-metrics/metrics-occurrences.md) dans le guide de l'utilisateur des composants.
* **Omniture :** Plus utilisé comme terme. organisation détentrice d'Adobe Analytics avant d'être acquise par Adobe en 2009.
* **Cheminement :** Voir Flux.
* **Page vue :** Type d'accès qui incrémente les pages vues. Voir [Pages vues](../components/c-variables/c-metrics/metrics-page-view.md) dans le guide de l'utilisateur des composants. Voir aussi Accès.
* **Persistance :** Concept abstrait pour les variables de conversion qui permet de lier une valeur de variable et un événement survenant sur des accès distincts. Voir aussi Expiration.
* **Appel du serveur principal :** Autre nom de demande d'image ou d'accès, utilisé principalement dans le contexte du balisage et de la facturation multi-suite. Lorsque le même accès est envoyé à plusieurs suites de rapports, la première suite de rapports est un appel serveur principal tandis que le reste est des appels serveur secondaires. Cette règle s'applique à tous les types d'accès, y compris le suivi des pages vues et le suivi des liens. Voir aussi Appels au serveur secondaire.
* **Règles de traitement :** Peut faire référence à :
   * Règles de traitement, moyen de modifier la collecte de données à l'aide de certaines règles dans la Console d'administration. See [Processing rules](../admin/admin/c-processing-rules/processing-rules.md) in the Admin user guide.
   * Règles de traitement des canaux marketing, ensemble de règles qui déterminent le canal marketing auquel un accès appartient. Voir Règles de traitement des canaux [marketing](../admin/admin/marketing-channels-admin.md) dans le Guide de l'utilisateur Admin.
* **Prop :** Voir Variable de trafic.
* **Rapport classé :** Format de rapport qui suit généralement une dimension avec une mesure. Ce type de rapport vous permet de voir les principaux éléments, tels que les pages les plus vues de votre site. Voir aussi Rapport de tendance.
* **Temps réel :** Affiche les variables configurées dès qu'elles sont collectées avec peu de latence. Reportez-vous à [la section Rapports](../admin/admin/realtime/realtime.md) en temps réel dans le guide de l'utilisateur Admin.
* **Suite de rapports :** Conteneur principal auquel vous envoyez des données. Tous les rapports d'Adobe Analytics référencent une suite de rapports.
* **Période variable :** Type de plage de dates relative qui change au fil du temps. Par exemple, un rapport montrant les 7 derniers jours peut être considéré comme une période variable. Voir aussi Plage de dates statique.
* **RSID :** Abréviation de l'identifiant de suite de rapports. Une suite de rapports présente un nom convivial et un identifiant de suite de rapports.
* **s. t () :** Nom de la fonction dans une bibliothèque appmeasurement qui envoie une demande d'image de page vue. Certaines bibliothèques appmeasurement utilisent `s.track()` à la place. Voir [s. t ()](../implement/js-implementation/function-t.md) dans le guide de l'utilisateur Implémentation.
* **s<span>.</span>tl () :** Nom de la fonction dans une bibliothèque appmeasurement qui envoie une demande d'image de suivi de lien. Certaines bibliothèques appmeasurement utilisent `s.trackLink()` à la place. Voir [s. tl ()](../implement/js-implementation/function-tl.md) dans le Guide de l'utilisateur Implémentation.
* **s_ code. js :** nom du fichier JavaScript utilisé dans les versions historiques d'Adobe Analytics. Le nom actuel du fichier JavaScript utilisé est appmeasurement. js.
* **Satellite :** Plus utilisé comme terme. Ancien nom du produit pour la gestion dynamique des balises.
* **Appel serveur secondaire :** Autre nom de demande d'image ou d'accès, utilisé principalement dans le contexte du balisage et de la facturation multi-suite. Lorsque le même accès est envoyé à plusieurs suites de rapports, toutes les suites de rapports après la première liste sont des appels serveur secondaires. Voir aussi Appels au serveur principal.
* **Segment :** Vous permet de vous concentrer sur un sous-ensemble spécifique de données. Voir [Segmentation](../components/c-segmentation/seg-overview.md) dans le guide de l'utilisateur des composants.
* **Conteneur de segments :** Partie d'un segment qui détermine le volume de données à placer. Les conteneurs peuvent être basés sur la page vue, la visite ou le visiteur. Voir [Segmentation](../components/c-segmentation/seg-overview.md) dans le guide de l'utilisateur des composants.
* **Sérialisation :** Voir Sérialisation d'événements.
* **Appel serveur :** Nom alternatif d'une demande d'image ou d'un accès, utilisé principalement dans le contexte de facturation.
* **Accès unique :** Visite où une dimension n'avait qu'une valeur unique. La visite peut avoir plusieurs accès, à condition qu'il n'y ait pas plusieurs valeurs uniques. Voir Accès [unique](../components/c-variables/c-metrics/metrics-single-access.md) dans le guide de l'utilisateur Composants. Voir aussi Rebond.
* **Sitecatalyst :** Plus utilisé comme terme. Ancien nom du produit pour Adobe Analytics.
* **Document de conception de solution :** Connue également sous le nom de référence de conception de solution ou SDR. Document interne géré par une organisation qui indique comment sont utilisées les variables personnalisées et la logique utilisée pour les renseigner. Voir [Création d'un document de conception de solution](../implement/prepare/solution-design.md) dans le Guide de l'utilisateur Implémentation.
* **Sous-relation :** Plus utilisé comme terme ; remplacé par des ventilations de dimension. Dans les versions précédentes d'Adobe Analytics, les sous-subrelations permettaient de ventiler les variables de conversion. Voir [Ventilation des dimensions](../analyze/analysis-workspace/components/dimensions/t-breakdown-fa.md) dans le guide de l'utilisateur Analyser.
* **Evénement de réussite :** Action suivie par un utilisateur. Votre organisation détermine les événements à suivre et les variables d'événement de réussite que vous utilisez pour le suivre. Voir [Evénements personnalisés](../components/c-variables/c-metrics/metrics-custom.md) dans le guide de l'utilisateur des composants.
* **Utilisateur pris en charge :** Voir Délégation du service clientèle.
* **Variable de trafic :** Connu sous le nom de props. Stocke une valeur personnalisée pour un accès unique. Les versions précédentes d'Adobe Analytics attribuaient une valeur unique aux props, mais les améliorations apportées à la plate-forme rendaient largement inutile les variables de trafic personnalisées. Adobe recommande généralement d'utiliser des variables de conversion (evars) dans la plupart des cas. Voir [Variables de trafic personnalisées](../components/c-variables/dimensionslist/reports-custom-traffic.md) dans le guide de l'utilisateur Composants.
* **Rapport de tendance :** Format de rapport qui montre généralement plusieurs plages de dates avec une mesure. Ce type de rapport vous permet de voir comment une mesure fonctionne au fil du temps. Voir aussi Rapport Classement.
* **Visiteur unique**: Représente le nombre de personnes uniques qui ont visité votre site. Un visiteur unique unique peut avoir plusieurs visites. Voir [Visiteur unique](../components/c-variables/c-metrics/metrics-unique-visitors.md) dans le guide de l'utilisateur Composants.
* **Suite de rapports virtuelle :** Conteneur virtuel de données qui référence une suite de rapports normale et permet d'affiner les données. Les données n'ont pas été envoyées à une suite de rapports virtuelle ; à la place, les données sont envoyées à une suite de rapports normale et une suite de rapports virtuelle est générée en dehors de ces données collectées. Voir Suites de rapports [virtuelles](../components/vrs/vrs-about.md) dans le guide de l'utilisateur des composants.
* **Visite :** Représente le nombre de sessions uniques qui ont eu lieu sur votre site. Voir [Visites](../components/c-variables/c-metrics/metrics-visit.md) dans le guide de l'utilisateur des composants.
* **Règle VISTA :** Logique personnalisée créée par Adobe à la demande d'un client pour copier, analyser ou filtrer les données côté serveur. Les règles VISTA envoient généralement des coûts supplémentaires. Voir aussi Règles de traitement.
* **Balise Web :** Voir Demande d'image.
