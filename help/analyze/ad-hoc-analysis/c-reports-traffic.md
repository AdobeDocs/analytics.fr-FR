# Rapports sur les sources de trafic

Les rapports de source de trafic vous donnent une vue détaillée des visiteurs et de la façon dont ils interagissent avec votre site Web.

## Rapports sur les sources de trafic {#concept_0F1772141E1345C5BCF63BE7C544C0CB}

Les rapports de source de trafic vous donnent une vue détaillée des visiteurs et de la façon dont ils interagissent avec votre site Web.

Les rapports de sources de trafic vous permettent :

* Analyser les aspects critiques du comportement des.
* Surveillez et comprenez les modèles de trafic.
* Déterminer le contenu le plus populaire du site.
* Segmentez les selon des critères mesurables.

**Persistance commune**

Dans [!UICONTROL Traffic Sources]ce cas, toutes les valeurs de rapport sont conservées et reçoivent du crédit jusqu’à ce qu’elles soient remplacées ou jusqu’à la fin de la visite, selon ce qui survient en premier. Auparavant, seuls les mots-clés et les domaines référents persistaient. Par exemple, si un visiteur recherche dans Google « DVD », qui l’amène sur votre site pour tout achat de 100 $, le rapport alloue un crédit de 100 $ au mot-clé « DVD » ainsi qu’au moteur de recherche Google. Cette fonctionnalité est inaltérable, quels que soient les paramètres d’[!DNL Admin Console].

## Type de référent {#concept_689E42D8F96C450DA41C7167C7388198}

En suivant et en enregistrant les sites de référence des visiteurs pour chaque visite, vous pouvez déterminer de quelle façon les visiteurs sont parvenus sur votre site pour chaque visite.

<!--

 -->

La liste ci-dessous définit les divers types de référents :

* *Les autres* de site Web sont enregistrés lorsque les cliquent sur un lien situé sur une page d’un autre site Web (non défini comme faisant partie de votre site) et arrivent sur votre site Web.
* *Les de moteur* de recherche sont enregistrés lorsque les utilisent un moteur de recherche pour accéder à votre site.
* *Les  tapées/marquées* d’un signet sont enregistrées

   * Si un accède à votre site par le biais d’un lien autre qu’un lien de navigateur (par exemple, dans un courrier électronique).
   * si un visiteur saisit directement l’URL de votre site dans son navigateur ;
   * Si un clique sur un lien HTML sur son disque dur personnel.
   * Si un accède à votre site en sélectionnant des signets de navigateur.

**Définitions**

Les lignes suivantes peuvent s’afficher lors de l’exécution de ce rapport :

**Dans votre site** : ces éléments sont des URL balisées par les filtres URL internes. Ces éléments ne sont pas comptabilisées comme instances de référence, mais elles sont visibles lors de la génération de rapports sur d’autres mesures.

**Aucun script** Java : Il n’y avait pas de code JavaScript, le type n’était donc pas identifiable (inconnu). Cela signifie qu’il n’y avait aucune information  de fournie par un client sur un navigateur, qui ne déclare pas être en mesure de prendre en charge Javascript. Elles ne sont pas comptabilisées comme &quot;instances &quot;, mais elles peuvent être vues lorsque vous  d’autres mesures.

**USENET (groupes de discussion)** : cela signifie que l’URL d’un référent commençait par `news://`. Dès lors, le lien de référence a été publié sur un groupe de discussion Usenet plutôt que sur une page Web.
